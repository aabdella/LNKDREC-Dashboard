# Leads

## Description
The Leads page is the outbound/company discovery workspace for finding hiring demand, reviewing results, and managing qualified leads.

## Available Features

### Feature: Multi-tab workflow
- The page is organized into tabs for search, history, qualified leads, and outreach-oriented work.
- Tabs separate discovery from follow-up.

### Feature: Search setup
- Operators can define country, boards, title, and other search criteria.
- This controls where and how lead discovery runs.

### Feature: Search results
- Search results return job opportunities and company demand signals.
- Results can be sorted and paginated.

### Feature: URL-backed state
- The page preserves key state in the URL.
- This makes it easier to share, restore, and revisit search context.

### Feature: Qualified leads
- Selected opportunities can become qualified leads.
- This transitions them from raw discovery into a structured pipeline.

### Feature: Enrichment
- Qualified leads can be enriched with contact data.
- This supports outbound action readiness.

### Feature: Search history
- Previous searches can be reviewed later.
- This reduces repeated work and helps compare discovery efforts.

## How features behave

### Multi-tab workflow
- Tabs separate phases of work so the page does not collapse everything into one noisy surface.
- Search is discovery; qualified leads and outreach represent later steps.

### Search setup
- Search configuration defines the discovery run.
- Operators can tune breadth and specificity depending on the market or role.

### Search results
- Results are meant for scanning, narrowing, and promotion into qualified lead workflows.
- Sorting and pagination exist to make larger result sets usable.

### URL-backed state
- URL state helps preserve working context across refreshes and navigation.
- This is especially valuable on a page with many filters and tabs.

### Qualified leads
- Marking a result as qualified is a meaningful transition from raw signal to actionable lead.
- This should be treated as a curation step, not a casual bookmark.

### Enrichment
- Enrichment expands a company/job signal into contactable information.
- This bridges search and outreach.

### Search history
- History exists to make lead generation repeatable and reviewable.
- It helps operators see what has already been searched and what outcomes were produced.

## Notes
- Leads is effectively a second product inside the platform.
- Future docs should add exact provider behavior, enrichment paths, and qualified lead status transitions.
