# Analytics

## Description
The Analytics page should surface performance, operational visibility, and business intelligence across the platform in a more deliberate way than workflow pages alone can provide.

## Available Features

### Feature: Funnel visibility
- Analytics should help explain conversion through sourcing, pipeline, and hiring stages.
- This makes workflow performance measurable rather than anecdotal.

### Feature: Delivery performance tracking
- The page should help show how delivery is performing against active demand.
- This is especially useful for recruiter operations and account health.

### Feature: Trend monitoring
- Analytics should make it easier to compare movement over time rather than just current snapshot state.

### Feature: Decision support
- The page should help operators and leadership decide where to focus effort, fix bottlenecks, or improve throughput.

## How features behave

### Funnel visibility
- Funnel metrics should reduce ambiguity around where candidates stall or progress.
- This is stronger when stage definitions are consistent across the platform.

### Delivery performance tracking
- Delivery tracking should connect demand-side records with recruiting output.
- It becomes more valuable when linked closely to jobs, client activity, and pipeline progression.

### Trend monitoring
- Trends should help distinguish one-off results from recurring patterns.
- This supports more grounded operational planning.

### Decision support
- Analytics should not exist just to display charts.
- Its job is to support action and clearer prioritization.

## Notes
- Some analytical capability already exists in adjacent pages like Sales, but this page should eventually become a more intentional reporting surface.
- Future docs should define which metrics are authoritative and which are exploratory.
